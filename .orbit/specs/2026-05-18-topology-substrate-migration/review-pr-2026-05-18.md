## Findings

### [LOW] Brownfield post-cleanup yaml is `{}\n` rather than an empty file
**Category:** edge-case
**Description:** When a brownfield config carries ONLY `docs.topology` (no other keys), the post-cleanup file content is `{}\n` rather than truly empty. This is functionally equivalent ‚Äî round-trips clean through `verify_all`, parses to a default `Config`, and the test `topology_setup_brownfield_strips_legacy_config` only asserts `docs.topology` is absent (not that the file is empty). Spec ac-05 says "the now-empty `docs:` block elided if it has no other keys" ‚Äî which is honoured for the `docs:` block specifically, but the surrounding empty `Config` serialises as `{}` rather than being deleted.
**Evidence:** `/tmp/orbit-review-brownfield/.orbit/config.yaml` after `topology setup`: `{}\n`. `orbit verify` reports clean. Test `topology_setup_brownfield_strips_legacy_config` (`verbs.rs:7378`) asserts `!new_text.contains("topology: docs/topology.md")` ‚Äî passes against `{}\n`.
**Recommendation:** No change. The behaviour is observably correct (clean round-trip, no drift) and the spec doesn't require the file to be deleted entirely. Documenting here for future readers tracking on-disk shape.

### [LOW] `audit_topology` drift codes diverge from spec narrative phrasing
**Category:** doc-drift
**Description:** Spec ac-02 names the drift-code vocabulary as the implementer's call (`stale_pointer`/`missing_entry`/`shape_drift` can be replaced, aliased, or extended). The implementation uses `stale_pointer` / `missing_entry` / `invalid_field` / `parse_failed`. `shape_drift` is documented in the skill table as a catch-all but is not emitted by `audit_topology` today (grep shows no `"shape_drift"` literal in `verbs.rs`). The topology SKILL.md table lists `shape_drift` as a category ‚Äî consumers reading the skill prose may expect it on the envelope, which it isn't.
**Evidence:** `plugins/orb/skills/topology/SKILL.md` line ~88: `| shape_drift | Catch-all for structural failures that don't fit the above |`. `verbs.rs::audit_topology` only emits `parse_failed` / `invalid_field` / `stale_pointer` / `missing_entry`.
**Recommendation:** No change required for merge ‚Äî the envelope contract is the source of truth, and the test suite asserts what's emitted. Either drop the `shape_drift` row from the skill table or have `audit_topology` emit `shape_drift` as the implementer-chosen alias when invariants outside the existing four buckets surface in future. Captured as a memo candidate.
