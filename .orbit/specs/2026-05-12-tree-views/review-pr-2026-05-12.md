## Findings

### [INFO] AC-02 missing-drift assertion is fixture-implicit, not directly tested
**Category:** test-gap
**Description:** AC-02 explicitly requires the verb to "surface any one-way drift" — where a card claims a spec the spec doesn't back-reference, or vice versa. The parity fixture (`populate_card_with_linked_spec`) tests only the healthy bidirectional case (`listed_on_card: true, back_referenced_by_spec: true`). The drift paths (one-way card→spec, one-way spec→card, missing spec file) exist in the code (`card_specs` in `verbs.rs:2109-2218`) and the response shape exposes both flags independently, but no parity test asserts the verb's behaviour when `back_referenced_by_spec=false` or `listed_on_card=false`.
**Evidence:**
- `crates/cli/tests/common/mod.rs:202-217` — only the healthy fixture; no drift fixture.
- `crates/core/src/verbs.rs:2168-2216` — code handles both directions but no test exercises the asymmetric cases.
**Recommendation:** Not a blocker. The drift logic is straightforward (two independent boolean flags in `CardSpecsEntry`) and the code path is short. Worth adding a follow-up fixture (`populate_card_with_one_way_spec_drift`) in a later spec to lock the behaviour against future refactors, but the AC is satisfied by the existing happy-path round-trip plus code inspection.

### [INFO] clippy warning on overview orphan filter
**Category:** style
**Description:** `cargo clippy` flags `verbs.rs:2308` — `reverse.get(*slug).map_or(true, Vec::is_empty)` could be `reverse.get(*slug).is_none_or(Vec::is_empty)`. Pre-existing-style code in this file uses `map_or`, so this is a stylistic nit rather than a real concern.
**Evidence:** `cargo clippy --release --all-targets 2>&1` — line `crates/core/src/verbs.rs:2308:38: this map_or can be simplified`.
**Recommendation:** Leave as-is; consistent with surrounding code, not worth a touch-up commit.

### [INFO] Unscoped graph render scales with substrate size
**Category:** performance / observation
**Description:** `orbit graph` without `--card` renders every card and every card-listed spec — on this repo (33 cards, ~70 specs) it emits ~6 KiB of mermaid. The AC explicitly permits this ("default render is permitted to exceed single-screen — serves the share-or-paste use case"); flagging for awareness, not as a defect.
**Recommendation:** None. Behaviour matches AC-04 contract.

## Honest Assessment

This is a clean, well-scoped implementation that ships exactly what the spec asked for. Five new verbs (`card.tree`, `card.specs`, `overview`, `graph`, `audit.drift`) wired on both CLI and MCP through a single Rust core, with parity tests covering happy paths and three error paths per verb (unknown id, malformed YAML via the drift surface, missing reference surfaced via `card.specs` status). The architectural commitments hold: every new verb is file-walking and read-only (no `state.db` reads in `core/src/verbs.rs` for the new functions — verified by inspection), the canonical error envelope is preserved, and the `Card::FIELDS` / `Spec::FIELDS` / `Choice::FIELDS` / `Memory::FIELDS` drift-prevention tests genuinely guard against schema drift (each test populates every field of every struct, so adding a new struct field without updating `FIELDS` will fail the assertion). The load-bearing AC-07 wire — `session prime` → `orbit overview` — is correctly threaded through the `SessionPrimeResult` struct and rendered by the CLI, so a fresh session does reach the synthesis layer in one step. The prose wires in `card` and `distill` SKILL.md are appropriately placed (after `card show`, before drafting respectively). Smoke-tested all five verbs end-to-end against this repo and they produce sensible output; idempotency holds (two consecutive `card tree --json` runs are byte-identical); cycle-safety holds (path-based visited set with `remove` on backtrack — the right semantic for tree expansion, not just-seen-set). One genuine gap worth noting: AC-02's drift-surfacing branches aren't directly tested — only the healthy bidirectional case. The code path is short enough and the AC's "surfacing" wording loose enough that this passes, but a follow-up fixture would lock the behaviour against regression. No blockers; merge clean.
